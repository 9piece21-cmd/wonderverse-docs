# 🚀 はじめに

---

## 1. Wonderverse|||へようこそ

Wonderverse は、ドラマ・ゲーム・EC など多様なシナリオに対応する、エンドツーエンドの AIGC クリエイションプラットフォームです。
アイデアひとつあれば、生成・編集・納品までを一箇所で完結できます。

> *ワンクリックで、アイデアを物語・映像・キャンペーンへ。*

### Wonderverse とは

従来のコンテンツ制作パイプラインでは、チームは十数のツールを切り替えながら作業します—台本はあるツール、画像生成は別のツール、吹替はまた別、編集と多言語版もそれぞれ。素材は散らばり、バージョンは追えなくなり、ツール間連携のコストはクリエイティブ作業そのものより重くなります。

Wonderverse は、このパイプライン全体を**ひとつのスタジオに収めます**。

- **基盤層**は 7 つの原子的 AI ツール：画像生成、動画生成、超解像強化、字幕翻訳、AI 吹替、横→縦変換、スマート消去。
- **中間層**は **無限キャンバス**—ノード型ワークフローで、これらのツールを自由に組み合わせ、保存し、再利用できます。
- **上位層**は 4 つの**すぐ使えるシナリオパック**：AI ドラマ、ゲームハイライト、EC コンテンツ、マーケティング動画。

[図：3 層アーキテクチャ図—下層に AI ツール、中層に無限キャンバス、上層にシナリオ]

### Wonderverse を選ぶ理由

#### 🎯 エンドツーエンド、断片的な継ぎ接ぎではない
台本から完成カットまで同じキャンバスで完結—ツール切り替えを 80% 削減。

#### 🌏 中・日・英のネイティブ対応
UI、吹替、字幕、ローカライズの全工程が中・日・英をネイティブサポート—第三者翻訳は不要です。

#### 🧩 シナリオはすぐ使える
ノード設計を覚える必要はありません—AI ドラマや EC コンテンツなどのテンプレートを開けば、5 分で初回完成カットが出せます。

#### 🔁 アセットとワークフローは蓄積される
チームのスタイル、キャラクター、ワークフローは再利用可能—使うほど速く、使うほど「あなたらしい」コンテンツに。

#### 🤝 チームコラボに最適
キャンバスは共有・再創作が可能、権限は明快、5〜50 人規模の中型コンテンツチームにフィットします。

- 🚀 **クイックスタート**—最初のプロジェクトを 5 分で完成

- 🎨 **無限キャンバス**—ノード、ワークフロー、アセット、コラボ
- 💡 **ベストプラクティス**—プロンプト、ローカライズ、スタイリングの応用テクニック

## 2. 誰のためのプロダクトか

Wonderverse は、規模化したコンテンツ制作を行う 3 種類の B 端チームに特化していますが、**P 端・C 端のクリエイターの皆様もぜひお試しください。**
以下のシナリオを参考に、ご自身に最適な入口を見つけてください。

### 🎬 AI ドラマ / ショート動画スタジオ

**典型的な課題**：台本から完成までのパイプラインが長い（1 話あたり 3〜7 日）。複数話制作でキャラクター / 画風がずれやすい。海外配信は吹替と字幕のやり直しが必要。

**推奨される入口**：
- 🎯 [AI ドラマ生成エージェント](#15--ai-ドラマ生成エージェント)—台本から完成カットまでのエンドツーエンド
- 🎙️ [AI 吹替・翻訳](#11-ai-吹替翻訳)—中・日・英の多言語版をワンクリックで
- 📐 [横→縦変換](#12-横縦変換)—横画面マスターからの賢い縦変換

**典型的なワークフロー**：台本を貼り付け → アセットライブラリでキャラクター固定 → AI 絵コンテ → バッチ生成 → 多言語出力

---

### 🎮 ゲーム会社

**典型的な課題**：試合 / ライブ素材は数時間単位、人の手では編集が追いつかない。複数プラットフォーム配信は何度も再編集が必要。新バージョンの宣材制作がリリース速度に追いつかない。

**推奨される入口**：
- 🎯 [ゲーム / 試合ハイライト](#16--ゲーム--試合ハイライト)—長尺素材を自動でショートに編集
- 🎬 [動画生成](#8-動画生成)—キャラクター / シーンのコンセプト映像
- 📝 [字幕生成 / 翻訳](#10-字幕生成--翻訳)—選手音声を字幕化、海外チャンネルへ直接配信

**典型的なワークフロー**：試合素材を取り込み → AI がハイライトを識別 → 自動編集 + 字幕 → 複数プラットフォーム仕様でバッチ出力

---

### 🛒 EC 企業（越境含む）

**典型的な課題**：商品撮影はコスト高で更新も遅い。モデル、シーン、スタイルの A/B テストを素早く回せない。海外サイトはローカライズしたモデル / コピー / 吹替が必要。

**推奨される入口**：
- 🎯 [EC コンテンツ制作](#17--ec-コンテンツ制作)—商品画像、モデル、シーンをワンクリック生成
- 🎨 [画像生成](#7-画像生成)—マーケティングメイン画像と KV
- 📢 [マーケティング動画](#18--マーケティング動画)—ショート動画広告のバッチ生成

**典型的なワークフロー**：商品画像をアップロード → カテゴリスタイル選択 → 多角度 / 多シーン / 多言語バージョンのバッチ生成 → 広告プラットフォームへ直接連携

---

### 🎨 P 端 / C 端クリエイター

**典型的な課題**：個人や小規模スタジオで大規模チームの予算はない。競争力のあるコンテンツを作りたいがツールのハードルが高く学習コストもかかる。アウトプットのペースが安定せず、インスピレーションが来た時に機材とスキルが追いつかない。

**推奨される入口**：
- 🎯 [無限キャンバス](#19-キャンバスを知る)—AI 機能を組み合わせて自分だけのワークフローを構築
- 🎨 [画像生成](#7-画像生成) / [動画生成](#8-動画生成)—単一ツールを直接呼び出し、学習コストゼロ
- 💡 [プロンプトのコツ](#24-プロンプトのコツ)—AI に本当に理解してもらう

**典型的なワークフロー**：アイデア → スタイル選択 → 単一ノードで出力 → 必要に応じて複数ノードで後処理

> 💡 **個人クリエイターでもプロ級のワークフローを使えます**—軽量な単一ツールから完全なワークフローまで、その時の必要に応じて。

## 3. 5 分間クイックスタート

Wonderverse は初めてですか？今日やりたいことに応じて、1 つのパスを選んでください—

> 🚀 **画像や動画をすぐ生成したい？** 「クリエイト」ワークステーションで 3 ステップ
> 🎨 **再利用できるワークフローを作りたい？** 「無限キャンバス」で 5 ステップ、ノードと接続をマスター

---

### パス 1 · 「クリエイト」で 3 ステップ

向いている人：初めて使う方、最速で結果を見たい方。

#### ステップ 1 · 「クリエイト」を開く
ログイン後、左サイドメニューから **「クリエイト」** をクリック。

#### ステップ 2 · 生成タイプとモデルを選ぶ
上部 Tab を **生成** に切り替え、**画像** または **動画** を選択。
Model 欄のドロップダウンで使いたいモデルを選びます（おすすめ：Gemini 3.0 / Kling 3.0）。

#### ステップ 3 · prompt を書いて実行
- 説明欄に prompt を書く
- 参考画像をアップロード（任意、最大 14 枚対応—モデルによる）
- アスペクト比、画質、バッチを選択
- **実行** をクリック—中央プレビュー欄に数十秒で結果が表示されます

> 編集したい？ **編集** Tab に切り替えて、超解像強化、字幕、吹替、横→縦変換、スマート消去を呼び出せます。

[図：クリエイトワークステーションのメイン画面]

---

### パス 2 · 「無限キャンバス」でワークフローを作る

向いている人：ノードの流れを理解したい方、複雑で再利用可能なパイプラインを作りたい方。

#### ステップ 1 · キャンバスを作成
ログイン後、左サイドメニューから **「無限キャンバス」** → **「新規空白キャンバス」** をクリック。
入った後、上部のプロジェクト名をクリックしてリネーム。

> 💡 **白紙から始めたくない？** 同じ入口に推奨テンプレート—ドラマ、EC、ソーシャルメディアの簡単なワークフローが用意されています。ワンクリックで開いて改変するのが最速です。

#### ステップ 2 · 最初のノードを追加（画像生成）
1. 左上の **「＋ ノード追加」** ボタンをクリックし、ノード選択パネルを表示
2. **「画像生成」** を選択、ノードがキャンバスに表示されます
3. ノードの入力欄に prompt を書く、例：
   > *「窓辺に座るブリティッシュショートヘアの猫、夕暮れの光が差し込む、シネマティックなトーン、35mm レンズ」*
4. **入力欄の左下** で使いたい画像生成モデルを選択（主要モデル対応）
5. ノードの **▶ 実行** をクリック

10〜30 秒後、ノード中央に生成された画像が表示されます。

#### ステップ 3 · 2 つ目のノードを追加（動画生成）
もう一度左上の **「＋ ノード追加」** をクリックし、**「動画生成」** を選択、画像生成ノードの右側に配置。

#### ステップ 4 · ノードを接続 · 画像を動画の先頭フレームにする
これがキャンバスのコア操作—**接続をドラッグ**。

1. 画像生成ノードの右側にある **出力ポート●** を押す
2. 動画生成ノードの左側にある **入力ポート●** へドラッグして放す
3. キャンバスに線が表示され、**上流の画像が下流の先頭フレームとして渡される**ことを示します

> 💡 これが「無限キャンバス」の真髄です—タイプが合う 2 つのノードなら、1 本の線でつなげます。

次に動画生成ノードで：
- 入力欄に**動き**の描写を書く：*「猫がゆっくり振り向き、髪が風になびく」*
- 入力欄左下で動画生成モデルを選択
- パラメータ：**アスペクト比 9:16 / 時間 5s**
- **▶ 実行** をクリック

[図：接続動作のデモ + ノードパラメータパネル]

#### ステップ 5 · プレビューとエクスポート
1. 60〜120 秒待って動画生成ノードに完成カットが表示
2. ノード中央をクリックでフルスクリーン再生
3. 満足したら、キャンバスの**右上**にある **「ケース出力」** ボタンをクリック—プロジェクトの全成果物（画像 + 動画）を一括エクスポート

🎉 これで最初のキャンバスワークフローが完成です—AI 画像から、それを先頭フレームとする AI 動画まで。

## 4. チームと権限

Wonderverse は 5〜50 人の中型コンテンツチーム向けに、**軽量だが実用的なコラボの仕組み**を設計しています。
素早くチーム作成、メンバー招待、権限割り当て、アセット共有を行い、制作プロセスを透明で制御可能に保てます。

### チーム作成

1. 個人ダッシュボード右上 → **チーム作成**
2. チーム名 + ロゴ（任意）を入力
3. チームプランを選択（[課金と請求書](#28-課金と請求書) を参照）

### メンバー招待

**チーム設定 → メンバー** から：

- メールで招待（最大 50 人を一括）
- 招待リンクを生成（有効期限 / 上限人数を設定可）
- OAuth で企業 SSO に接続〔Enterprise〕

### ロールと権限

| ロール | プロジェクト権限 | アセット権限 | 課金権限 |
|---|---|---|---|
| **🛡 管理者 Admin** | 作成 / 削除 / 全アクセス | 全読み書き | ✅ |
| **🎨 クリエイター Editor** | 自分 + チーム共有を作成 / 編集 | 自分が作ったものを読み書き | ❌ |
| **👀 オブザーバー Viewer** | 共有されたプロジェクトを読み取り専用 | 読み取り専用 | ❌ |
| **🤝 ゲスト Guest** | 単一プロジェクトの一時的な読み取り専用 | ❌ | ❌ |

### プロジェクト共有

各プロジェクトに 3 段階の公開設定：

- 🔒 **プライベート**—作成者のみ可視
- 👥 **チーム共有**—チームメンバーがロールに応じてアクセス
- 🌐 **公開リンク**—リンクを持つ誰でも読み取り専用でアクセス可（外部レビューに最適）

### アセットライブラリ

チームのキャラクターライブラリ、音声ライブラリ、スタイルライブラリ、テンプレートライブラリはすべて**アセットライブラリ**に集約されます：

- メンバーが保存したアセットは、ワンクリックで**チームアセット**に昇格して全員が呼び出せます
- アセットはバージョン管理とロールバック対応
- 管理者は **アセットライブラリ → ガバナンス** で承認フローを設定可（Enterprise）

### データセキュリティ〔Enterprise〕

- プライベートデプロイ / VPC ネットワーク分離〔要確認〕
- 国密準拠 / GDPR 準拠サポート〔要確認〕
- 操作監査ログ、全行動を追跡可能〔要確認〕

## 7. 画像生成

テキスト記述から画像を生成します。Wonderverse の画像生成モデルは、**EC 商品**、**ドラマスタイル**、**マーケティング KV** の 3 つのシナリオに特化最適化されています。

### 機能概要

- ✅ 画像生成（純 prompt）
- ✅ 画像から画像（参考画像 + prompt）
- ✅ 多画像融合（最大 14 枚の参考画像対応—モデルにより異なる）
- ✅ 多種類の組み込みスタイル（日本アニメ、3D リアル、水墨、サイバーパンク、Q 版など）

### 入力

| 入力項目 | 説明 |
|---|---|
| **Prompt（必須）** | テキスト記述、最大 1000 文字 |
| **参考画像（任意）** | 最大 14 枚、具体的な枚数はモデルによる |
| **負の Prompt（任意）** | 含めたくない要素 |

### パラメータ

| パラメータ | 選択肢 | デフォルト |
|---|---|---|
| **スタイルプリセット** | 日本アニメ / 3D リアル / 水墨 / サイバーパンク / Q 版 / 写真リアル / カスタム | 写真リアル |
| **アスペクト比** | 1:1 · 4:3 · 3:4 · 16:9 · 9:16 · カスタム | 1:1 |
| **解像度** | SD / HD / Ultra HD | HD |
| **シード Seed** | 任意の整数；同 prompt + 同 seed = 同じ画像 | ランダム |
| **バッチ** | 1× / 2× / 4× | 1× |

### Prompt 速習

高品質な prompt は通常 5 つの要素を含みます：

```
[被写体] + [動作 / 状態] + [環境 / 時間] + [ビジュアルスタイル] + [カメラ / ライティング]
```

**例**：
> 着物姿の少女（被写体）が桜の木に寄りかかる（動作）+ 京都の春の朝（環境）+ 浮世絵スタイル（スタイル）+ 望遠で背景ボケ（カメラ）

詳細は [プロンプトのコツ](#24-プロンプトのコツ) を参照。

### よくある質問

**Q：参考画像は最大何枚？**
A：最大 14 枚、具体的な枚数はモデルによります。

**Q：同じキャラクターを固定して生成できる？**
A：可能です。[パーソナルアセット](#22-パーソナルアセット) でキャラクターをアセットカードとして保存すれば、毎回同じ姿で生成されます。

**Q：商用著作権はどうなる？**
A：有料ユーザーは生成画像の商用権を保有〔詳細は ToS 参照〕。

## 8. 動画生成

テキスト記述（先頭 / 先尾フレームも任意）から動画を生成します。
Wonderverse の最もコアな動画制作エンジンであり、ドラマ / ハイライト / マーケティング動画シーンの原動力です。

### 機能概要

- ✅ 純動画生成
- ✅ 画像から動画（単一画像を先頭フレームに）
- ✅ 先頭・先尾フレームから動画（2 枚を先頭・末尾とし、AI が中間モーションを補完）
- ✅ 動画延長（既存動画の末尾から継続生成）
- ✅ 音画同時出力（一部モデル対応、例：Kling 3.0 Omni、音声付き動画を直接生成）

### 入力

| 入力項目 | 説明 |
|---|---|
| **Prompt（必須）** | ショット内容を記述、最大 1500 文字 |
| **参考画像（任意）** | モデルにより異なる、最大 4 枚 |
| **先頭フレーム（任意）** | 1 フレーム目を固定 |
| **先尾フレーム（任意）** | 先頭フレームとペアで、AI が中間モーションを補完 |
| **負の Prompt（任意）** | 含めたくない要素 |

### パラメータ

| パラメータ | 選択肢 | デフォルト |
|---|---|---|
| **時間** | 4s · 5s · 6s · 8s · 10s · 12s（モデルにより異なる） | 5s |
| **アスペクト比** | 16:9 · 9:16 · 1:1（モデルにより異なる） | 16:9 |
| **解像度** | 720p · 1080p | 720p |
| **スタイルプリセット** | 画像生成と同じ | リアル |

### Prompt の注意点

動画 prompt と画像 prompt の違い：**動きを必ず描写すること**。

```
✅ 推奨：カメラがゆっくり前進、少女が振り向き微笑む、髪が風に揺れる
❌ 非推奨：微笑む少女
```

詳細は [プロンプトのコツ · 動画編](#24-プロンプトのコツ) を参照。

> 💡 より高画質が必要？動画生成後に「編集」カテゴリの「超解像強化」につないで、1080P / 2K / 4K 出力を取得できます。

## 9. 超解像強化

ぼやけた、低解像度、ノイズの多い動画や画像の画質を向上させます。
**古い素材の修復**、**ユーザーアップロードの低品質素材の改造**、**生成結果の二次精修**の 3 つのシナリオに最適。

### 機能概要

- ✅ 動画強化（4 段階の出力：720P / 1080P / 2K / 4K）
- ✅ 画像強化（3 種類の強化モード：標準 / 人物 / 暗光）
- ✅ コンテンツタイプの自動適応（ドラマ / 実写 / 通用）
- ✅ 小顔最適化（動画専用、顔が小さい時に五官をよりよく復元）

### 入力

| タイプ | 対応フォーマット |
|---|---|
| 🎬 **動画** | MP4 / MOV / WebM、推奨 ≤ 5 分 |
| 🖼️ **画像** | JPG / PNG / WebP |

### 動画パラメータ

#### コンテンツタイプ（画面の性質で選択）

| 選択肢 | 説明 |
|---|---|
| 🎭 **ドラマ / アニメ** `おすすめ` | アニメ風、ドラマシーン向け最適化 |
| 👤 **実写** | 実写出演、肌色 / ディテール最適化 |
| 🎨 **通用** | 迷ったらこれ、各種画面に対応 |

#### 強化レベル（出力解像度と精修強度を決定）

| レベル | 出力 |
|---|---|
| ⚡ **クイック修復** | 720P |
| 🎯 **標準修復** | 1080P |
| ✨ **究極強化** | 2K |
| 🎬 **シアター級** | 4K |

#### 小顔最適化

トグル。顔が小さい時に五官をよりよく復元します。

### 画像パラメータ

#### 強化モード

| 選択肢 | 説明 |
|---|---|
| 📐 **標準** `デフォルト` | 大部分の画像に対応 |
| 👤 **人物** | 実写 / 自撮り、顔強化を有効化 |
| 🌃 **暗光** | 夜景 / 室内が暗い、明るさを上げる |

### 典型的な使い方

| シナリオ | 推奨設定 |
|---|---|
| ドラマの古い素材修復 | コンテンツ「ドラマ / アニメ」+ 究極強化 2K（またはシアター級 4K） |
| 実写ショートドラマ / 配信録画 | コンテンツ「実写」+ 標準修復 1080P + 小顔最適化 ON |
| 迷ったら / 混合画面 | コンテンツ「通用」+ 標準修復 1080P |
| 古い写真 / ぼやけた画像 | 強化モード「標準」（画像） |
| 自撮り精修 | 強化モード「人物」（画像） |
| 夜景・暗トーン写真 | 強化モード「暗光」（画像） |

## 11. AI 吹替・翻訳

動画内である言語を話す人物を、別の言語で原音色そのままに話させます。
30 ソース言語の相互翻訳、音声クローン + 字幕同期焼き込み。

### 機能概要

## 15. 🎬 AI ドラマ生成エージェント

AI ドラマ制作に必要なキャラクター抽出、アセット構築、絵コンテ設計、動画生成、編集出力—
そのすべてをエンドツーエンドのエージェントに委ねます。あなたが行うのは 2 つだけ—**ストーリーを与え、完成品を選ぶ**。

> 📍 製品左サイドメニューの「**AI ドラマ**」入口に対応。
> 対象：AI ドラマ / ショート動画会社、IP スタジオ、海外配信チーム。
> 典型的な工期：1 話 60〜180 秒、**台本から初版完成カットまで約 30 分**。

### あなたの何を解決できるか

| 従来のドラマパイプラインの課題 | Wonderverse のアプローチ |
|---|---|
| 台本分解は手作業の絵コンテ、1 話 3〜5 日 | エージェントが**台本を自動解析**、5 分で絵コンテ表 |
| キャラクター / シーンが話を跨いでブレ、画風が漂流 | **アセットライブラリで固定**、キャラ・声・シーンが全話で一貫 |
| 吹替の外注は高く、修正は遅く、多言語化は困難 | **AI 吹替 + クローン**、中・日・英をワンクリック切替 |
| 出力規格が煩雑（横画面 / 縦画面 / 長尺・短尺） | **複数アスペクトを一括出力**、全プラットフォーム対応 |

### 完全ワークフロー（5 ステップ）

[図：5 ステップドラマワークフロー]

### ステップ 1 · 台本を入力

**シナリオ → 🎬 AI ドラマ生成エージェント** に入ると、メイン画面はチャットボックスです。台本を提供する方法は 2 つ：

#### オプション A · 既存台本を貼り付け
Word、TXT、プレーンテキストの台本を直接貼り付け。長文対応（**1 回最大 10 万字**）、エージェントが話数で自動分割。

#### オプション B · AI に書いてもらう
ストーリーのログラインを 1 文入力、例：
> *「東京渋谷、記憶喪失の元プロの殺し屋がコンビニで自分の娘と出会う。」*

エージェントがあなたの設定に従って完全な台本を生成します。

#### 設定項目
| 設定 | 説明 |
|---|---|
| 🎨 **スタイル** | 日本アニメ、3D リアル、水墨、サイバーパンク、Q 版など多種類組み込み |
| 📺 **話数** | 1 〜 100 話 |
| ⏱️ **1 話の時間** | 60s / 90s / 120s / 180s |
| 🎞️ **アスペクト比** | 9:16 縦 · 16:9 横 · 1:1 正方形（複数選択可） |
| 🌏 **言語** | 中国語 / 日本語 / 英語 |

[図：台本入力ダイアログ + 設定パネル]

### ステップ 2 · アセットライブラリを構築

台本送信後、エージェントが自動的に 3 種類のアセットを抽出：

#### 🧑 キャラクター
登場人物ごとに**ビジュアルカード + 性格カード + 声カード**を自動生成。以下が可能：
- キャラクターのビジュアルを差し替え（参考画像アップロード / 再生成 / アセットライブラリから選択）
- **実在の人声をクローン**（30 秒のクリーン人声サンプルをアップロード）
- 組み込み音声ライブラリから選択（中・日・英の多言語声含む）

#### 🏞️ シーン
ロケ地ごとに自動グループ化し、メインビジュアル生成。調整可能：
- 時間（早朝 / 夕方 / 深夜）
- 天候（晴 / 雨 / 雪 / 霧）
- ライティングの雰囲気（暖 / 寒 / ハイコントラスト）

#### 🎭 小道具
重要な小道具（武器、思い出の品、キーアイテム）を個別管理、複数ショット間の一貫性確保。

> 💡 **アセットは蓄積されます**—今話で作ったキャラクター・声・シーンをチームライブラリに保存すれば、次作品でそのまま再利用できます。

### ステップ 3 · AI 絵コンテ

エージェントが台本を段落ごとにショットに切り分け、**絵コンテ表**を出力。各ショットには：

| 項目 | 例 |
|---|---|
| ショット番号 | 01-007 |
| 画角 | 中景 / クローズアップ / ワイド |
| キャラクター | 主役 A + 通行人 B |
| シーン | 渋谷コンビニ · 雨夜 |
| アクション | 主役 A がゆっくり振り返り、視線がレジに落ちる |
| セリフ | 「やっぱり…君だったか。」 |
| 時間 | 3.2 秒 |

各行を**個別に編集可**—文案の書き換え、画角変更、時間調整、注釈追加。

### ステップ 4 · 動画を生成

絵コンテを確定したら **生成** をクリック：

- **単ショット生成**：ショットごとにプレビュー、気に入らなければ再ロール
- **バッチ生成**：1 話まとめて実行、エージェントが自動キュー
- **全シリーズ生成**：100 話レベルでバックグラウンドバッチ出力（**夜間バッチに最適**）

> 動画生成は Wonderverse 動画エンジンベース、720p / 1080p 対応；ショットごとに 1× / 2× / 4× の再ロール回数設定可。

### ステップ 5 · 編集と出力

動画生成完了後、編集卓に入ります：

- **基本編集**：トリミング、結合、トランジション、字幕、BGM
- **複数アスペクト出力**：横・縦・正方形を一括出力、自動再構図（横→縦変換ベース）
- **多言語版**：ターゲット言語をチェック、AI 吹替 + 字幕翻訳を自動呼び出し、ローカライズ版をワンクリック生成
- **出力フォーマット**：MP4 (H.264 / H.265)、MOV、最大 1080p

> 🚧 **編集機能は継続的に進化中**—高度なタイムライン、マルチトラックミックス、AI 自動編集などの機能が予定されています。

### 完全な例

> **シナリオ**：日本の EC ブランド「花鳥集」が新作香水ラインの宣伝に 12 話の縦画面ドラマを作りたい、ターゲットは TikTok 日本リージョン。

### よくある質問

**Q：台本が長すぎるとカットされる？**
A：1 回最大 10 万字、超過分は章節ごとに複数プロジェクト管理に自動分割。

**Q：全シリーズでキャラの一貫性を保証できる？**
A：アセットライブラリでキャラビジュアル + 声 ID を固定、全シリーズが同じアセットを呼び出すため一貫性が確保されます。

A：できます。各ステップは独立ノード、修正すると**下流のみ再実行**、すでに生成された素材は無駄になりません。

**Q：音声クローンに必要なサンプル時間は？**
A：30 秒のクリーンな中立人声で OK（BGM なし / エコーなし）。

| 格闘 | 完璧なコンボ / KO / カウンター |
| カスタム | キーワードやタイムスタンプで手動マーキング |

各ハイライトモーメントが自動タグ付け、タイムラインで直接追加・削除可。

### ステップ 3 · 自動編集

エージェントがリズムに合わせて切り貼り：

- **リズムライブラリ**：燃え / 速 / 緩 / エモーショナル（組み込み BGM ライブラリ）
- **トランジション**：ハードカット / フラッシュホワイト / スライド / ズーム
- **OP / ED**：クラブ / ブランド専用パッケージにバインド可
- **長さ計画**：30s / 60s / 90s / カスタム

### ステップ 4 · 字幕 + 吹替

- 選手 / 解説の原音を字幕に自動認識
- 🚧 AI 解説（試合データから燃え系ナレーションを自動生成）—近日リリース
- 多言語字幕焼き込み + 海外言語吹替

### ステップ 5 · 複数プラットフォーム出力

| プラットフォーム | 推奨規格 |
|---|---|
| 抖音 / TikTok | 9:16 · 1080p · 30-60s |
| YouTube Shorts | 9:16 · 1080p · 60s |
| Twitter / X | 16:9 · 720p · 90s |
| WeChat 動画チャンネル | 9:16 · 1080p · 60s |
| Bilibili | 16:9 · 1080p · 任意 |

1 回編集 → ターゲットプラットフォームをチェック → 対応規格でバッチ出力。

| ステージ | 操作 | 所要時間 |
|---|---|---|
| 取り込み | 90 分の録画（約 800 MB）をアップロード | 3 分 |

## 19. キャンバスを知る

**無限キャンバス** は Wonderverse のメイン制作戦場です。
これまでの「台本 → 素材 → 編集」はバラバラのツールページで起きていました；
ここでは、それらが**無限に広がる同じキャンバス上**に集約され、あなたがつなぎ、AI が引き継ぎます。

[図：無限キャンバスのフルスクリーンビュー]

### 従来の制作ツールとどう違う

| 観点 | 従来のツール | 無限キャンバス |
|---|---|---|
| **基本単位** | ファイル、レイヤー、タイムライン | **ノード**（1 つの AI 機能 = 1 つのノード） |
| **制作思想** | 直線的フロー：A が終わって B へ | **ネットワーク協調**：複数ブランチ並行、結果即時フィードバック |
| **再利用方法** | テンプレートのコピペ | **ワークフローを保存**、次回はワンクリックで再実行 |
| **コラボ方法** | ファイルのやり取り | キャンバス共有、誰でも全フローが見える |
| **トライ&エラーコスト** | 1 箇所変えると全チェーン再作業 | 1 箇所変えると下流のみ再実行 |

### キャンバスの 3 つのコア概念

#### 🧩 ノード Node
キャンバス上の最小操作単位。各ノードが 1 つの AI 機能をカプセル化—例えば「画像生成」「スマート消去」「AI 吹替」。
ノードには入力と出力があり、LEGO のように積み重ねられます。

#### 🔗 ワークフロー Workflow
複数のノードを線で組み合わせると**ワークフロー**になります。
ワークフローは保存、命名、再利用、共有可能—チームの真の生産力資産。

#### 📦 アセット Asset
キャンバス上で蓄積された成果—素材、キャラ、スタイル、声、テンプレート—は
**アセットライブラリ**に集約。アセットは任意のノードから呼び出せ、チームのスタイルがすべてのプロジェクトに継承されます。

### キャンバスの基本レイアウト

[図：キャンバスメイン画面レイアウト]

| エリア | 機能 |
|---|---|
| **左ツールバー** | ノードライブラリ · ワークフローテンプレート · アセットクイックアクセス |
| **中央キャンバス** | ドラッグ、接続、ズーム、パン；無限サイズ対応 |
| **右アセットパネル** | 現在のプロジェクトの素材、キャラ、声、スタイル |
| **トップバー** | プロジェクト名 / 元に戻す・やり直し / チームコラボ / グローバル設定 |
| **ボトムバー** | ズーム比率 / 現在の座標 / グローバル検索 |

### キャンバスでできること

#### 1. エンドツーエンドパイプラインを構築
「画像生成 → スマート消去 → 動画生成 → 字幕翻訳」を 1 つのフローに、台本を入力すると多言語完成カットを自動出力。

#### 2. 複数ブランチで並行トライ
同じ入力を 3 つの異なるスタイルの「画像生成」ノードに分岐、3 バージョンの結果を並べてプレビュー、最も満足のいくものを下流へ。

#### 3. 成功フローをテンプレートとして蓄積
チームで最も回しやすいワークフローをテンプレートとして保存、新メンバーが箱開けですぐ使え、暗黙知の継承問題とおさらば。

#### 4. 共有と二次創作
キャンバスリンクを出力、外部チームが同じキャンバスを開いて閲覧、コピー、改変。
全員の改善が見える化、**透明なコラボ**がメールのやり取りに代わります。

## 20. ノードを知る

ノード（Node）は Wonderverse キャンバスの最小操作単位。
**1 つのノード = 1 つの AI 機能**—話せて、絵が描けて、動画が編集できる職人のような存在です。
ノードを覚えれば、無限キャンバスの 90% の操作をマスターしたことになります。

### ノードの解剖図

[図：ノード解剖]

| 部位 | 機能 |
|---|---|
| **出力プレビュー** | 現在のノードが生成した結果—画像、動画、テキスト |
| **入力欄** | AI に指示を出す / 素材アップロード / 上流ノード出力を参照 |
| **ショートカット** | 組み込みのサンプル prompt、クリックで自動入力 |
| **パラメータ ⚙** | モデル、スタイル、サイズ、強度などの調整可能パラメータ |
| **実行回数** | 1× / 2× / 4×—同じ指示を「ガチャ」のように複数回試す |
| **実行ボタン ▶** | 現在のノードを実行 |
| **入力 / 出力ポート** | 上流から受信 / 下流に出力 |

### ノードの種類

Wonderverse には 7 大カテゴリのノードが組み込み、すべての基盤 AI 機能をカバー：

| ノードタイプ | 入力 | 出力 | 用途 |
|---|---|---|---|
| 🎨 **画像生成** | テキスト prompt | 画像 | クリエイティブ素材、絵コンテ、ポスター |
| 🎬 **動画生成** | テキスト prompt（先頭フレームも可） | 動画 | ショートクリップ、シーン素材 |
| 🔍 **超解像強化** | 画像 / 動画 | 高画質画像 / 動画 | 2× 拡大 · ノイズ除去 · カラーグレーディング |
| 📝 **字幕生成 / 翻訳** | 動画 | 多言語字幕ファイル / 焼き込み済み動画 | 多言語ローカライズ |
| 🎙️ **AI 吹替** | テキスト + 音声 | 音声 | 多言語ナレーションとセリフ |
| 📐 **横→縦変換** | 横画面動画 | 縦画面動画 | プラットフォーム対応 |
| 🩹 **スマート消去** | 動画 / 画像 + 選択範囲 | クリーン版 | ロゴ / 通行人 / 物の除去 |

> 💡 後続でさらに多くのノードが追加予定（音楽生成、長尺動画スティッチング等）、すべて同じインタラクションロジックを共有。

### ノードの入力：3 つの来源

#### 1. 直接入力
手動で指示を書くか、ファイルをアップロード。最も一般的な使い方。

#### 2. 上流ノードを参照
**線で上流ノードの出力ポートを接続**、下流が自動的に上流の結果を受け取ります。
> 例：「画像生成」ノードを「超解像強化」ノードに接続、前者が画像出力、後者が自動拡大。

#### 3. アセットライブラリを呼び出し
右側のアセットパネルからキャラ、声、スタイルをドラッグ、ノードがそれを参考入力として扱います。
> 例：「主役 A ビジュアルカード」を「画像生成」ノードにドラッグ → 生成画像はすべて同じキャラビジュアル。

### ノードをより使いこなす 2 つのコツ

#### ⚙ パラメータパネルで詳細を調整
デフォルトパラメータは 80% のシーンに適しますが、⚙ パネルで以下を調整可：
モデルバージョン、スタイル強度、アスペクト、シード、ガイダンス係数等。

#### 🔁 実行回数で「ガチャ」を回す
同じ指示を 4 回試して、最も気に入ったものを選ぶ。
手動で繰り返すより時間節約、トークンも節約。

### ノードの接続ルール

[図：ノード接続例]

- 🟢 **タイプが一致するポートのみ接続可**：画像 → 画像、動画 → 動画、テキスト → テキスト
- 🔀 **1 つの出力を複数の下流に接続可**：分岐比較、並行トライ
- 🔂 **上流ノードを修正 → 下流が自動無効化**：素材バージョン乱れを防ぐ
- ⏯ **全チェーンをワンクリックで実行**：末端ノードの ▶ をクリックすると、システムが線に沿って上流まで遡って実行

## 21. ワークフローを作る

単一ノードを**完全な生産ラインのチェーン**につなぐ—これがワークフロー（Workflow）です。
作ったワークフローは保存、命名、再利用、共有可能、Wonderverse チームの最も重要なアセット形態のひとつ。

### 3 ステップで最初のワークフローを構築

#### ステップ 1 · ノードをドラッグ
左側のノードライブラリから必要なノードをドラッグ、各ノードをキャンバス上の独立した位置に配置。
> 💡 **おすすめ**：左から右へ、上から下へ並べると、流れが直感的になります。

#### ステップ 2 · ポートを接続
上流ノードの**出力ポート**を下流ノードの**入力ポート**にドラッグ：

[図：3 ノード接続]

タイプが一致しない場合は赤色で警告、接続できません。

#### ステップ 3 · ワンクリック実行
末端ノードの ▶ ボタンをクリック、システムが**線に沿って上流まで遡って**：
- 未実行の上流ノードを自動実行
- すでに結果がある未変更ノードは直接再利用（リソースを無駄にしない）
- 各ノードの実行ステータスをリアルタイム表示：⏳ 待機 → 🟡 実行中 → 🟢 完了 / 🔴 失敗

### 上級 · 分岐と合流

ワークフローは直線である必要はなく、分岐と合流が可能：

#### 分岐（一対多）
1 つの出力を複数の下流ノードに接続可能：
> 例：1 枚の原画像を「スタイル A 画像生成」と「スタイル B 画像生成」に同時投入、並べて比較

#### 合流（多対一）
一部のノードは複数入力の合流をサポート：
> 例：3 つの絵コンテ動画 → 動画結合ノード → 完全なショート〔ノード対応状況は要確認〕

### 保存と再利用

ワークフロー実行後：

1. トップバーの **ワークフローとして保存** をクリック
2. 名前、カバー、説明を設定
3. 公開設定を選択：プライベート / チーム / 公開

次回呼び出し：
- **テンプレートから新規プロジェクト** → このワークフローベースで開始
- **新しいキャンバスにドラッグ** → 既存プロジェクトにこのフローを埋め込む
- **API 呼び出し**〔Enterprise〕→ ワークフローをサービスとして業務システムに接続

### ワークフローテンプレートライブラリ

Wonderverse は数十種のワークフローテンプレートを組み込み、シーン別に分類：

- 🎬 ドラマ絵コンテフロー（日本アニメ / リアル / 水墨のバリエーション含む）
- 🎮 試合ハイライト編集フロー
- 🛒 EC メイン画像バッチフロー
- 📢 マーケティング動画フロー
- 🌏 多言語ローカライズフロー

テンプレートをワンクリックで自分のキャンバスに fork、改変可能。

## 23. 共有とコラボ

Wonderverse のキャンバスは**コラボのために生まれた**—
「メールでファイルをやり取りする」非効率はもうない、全員が同じキャンバス上でフローを見て、編集して、再利用できます。

### 3 つの共有方法

#### 1. チーム内共有
プロジェクト設定 → チームに共有 → 各メンバーのアクセス権限を設定：
- **編集可**：完全修正可
- **コメント可**：注釈は残せるが修正不可
- **読み取り専用**：見るだけ

#### 2. 公開リンク共有

| 選択肢 | 説明 |
|---|---|
| **公開設定** | 閲覧のみ / コピー可（相手のアカウントにコピーして編集継続） |
| **有効期限** | 24 時間 / 7 日 / 永久 |
| **パスワード保護** | 任意 |
| **ウォーターマーク** | 「by チーム名」ウォーターマーク追加可 |

#### 3. 二次創作（Fork）
相手があなたの公開キャンバスを開いた後、**自分のワークスペースにコピー** をクリック—自分のアカウントに独立したコピーを取得して改変可能。

### リアルタイムコラボ

複数人が同じキャンバスを同時に開いた時：
- 各メンバーのカーソル / 現在選択中のノードをリアルタイム表示
- 誰がどのノードを変更したかアバターマーク付き
- コメントは具体的なノードにアンカー、文脈ある対話

### コメントとレビュー

#### ノードコメント
ノードを右クリック → コメント追加。コメントで指定メンバーを @ 可、メール / アプリ内通知が届きます。

#### プロジェクトレビュー
プロジェクトを「ゲスト（Guest）」ロールに共有、相手は閲覧 + コメントのみ可、修正不可—
クライアント / 上司に初版を見せるのに最適。

### コラボ中のバージョン管理

- 自動保存：30 秒ごと
- 履歴バージョン：右上の ⏰ で任意時点のキャンバススナップショット表示
- ワンクリックロールバック：履歴バージョンを選択 → 復元

### 部門間コラボ

協業相手も Wonderverse 上にいる場合：
- **部門間共有** で単一プロジェクトを一時的に相手に渡す
- アセット / ワークフローを単独パッケージとして「出力」、相手に送ってインポートしてもらう

## 24. プロンプトのコツ

プロンプト（Prompt）はあなたと AI の**共通言語**です。良い prompt は生成品質を 60 点から 90 点に押し上げます。
本章では 3 種類のノードの prompt 速習公式 + 落とし穴チェックリストを提供。

### 普遍原則

| 原則 | 説明 |
|---|---|
| **具体 > 抽象** | 「夕暮れの金色の陽光」 > 「いい光」 |
| **数値化 > 形容** | 「35mm レンズで被写界深度ボケ」 > 「映画感がある」 |
| **少ない方が多い** | キーワード 6〜12 個がベスト、多すぎるとかえって薄まる |
| **構造化** | 「被写体 + 動作 + 環境 + スタイル + カメラ」の順で書く |
| **否定をうまく使う** | negative prompt で不要要素を排除 |

### 画像生成 · prompt 公式

```
[被写体] + [動作 / 状態] + [環境 / 時間] + [ビジュアルスタイル] + [カメラ / ライト]
```

**例**：

| 観点 | 例 |
|---|---|
| 被写体 | 着物姿の少女 |
| 動作 | 桜の木に寄りかかる |
| 環境 | 京都の春の朝 |
| スタイル | 浮世絵スタイル |
| カメラ | 望遠で背景ボケ、柔らかな光 |

合体：
> *「着物姿の少女、桜の木に寄りかかる、京都の春の朝、浮世絵スタイル、望遠で背景ボケ、柔らかな光」*

#### スタイルキーワード早見表
- **写真リアル**：`写真リアル、35mm、シャープフォーカス、自然光`
- **日本アニメ**：`アニメ風、セルシェーディング、鮮やかな色彩、表情豊かな目`
- **3D リアル**：`3D レンダリング、Octane、柔らかな影、サブサーフェススキャタリング`
- **サイバーパンク**：`サイバーパンク、ネオン、雨、反射、未来感`
- **水墨**：`水墨画、中国伝統、ミニマル、モノクロ`

### 動画生成 · prompt 公式

動画 prompt と画像 prompt の違いはコア要素が 1 つ多いこと：**動き**。

```
[被写体 + 状態] + [動作の方法] + [カメラの動き] + [環境の雰囲気] + [スタイル]
```

**例**：

| 観点 | 例 |
|---|---|
| 被写体 | コンビニに座る老人 |
| 状態 | 新聞を読む、集中した表情 |
| 被写体の動き | ゆっくりページをめくる、たまに顔を上げる |
| カメラの動き | カメラが遠景から中景にゆっくり前進 |
| 環境 | 雨夜、ガラス窓の反射、ネオンの逆光 |
| スタイル | 日本映画感、温かい色調 |

#### 動画 prompt の落とし穴

| ❌ 非推奨 | ✅ 推奨 |
|---|---|
| 微笑む少女 | 少女がゆっくり振り向き、微笑みを見せる |
| 風景がきれい | カメラが横にパン、雪山と湖を捉える |
| 老人が本を読んでいる | 老人がページをめくる、たまに車の音 |

### 吹替 · prompt のコツ

テキスト自体が prompt ですが、いくつかの細部で効果が飛躍：

#### 1. 感情をタグ付け
```
[喜び]「やっぱり、ここにいたんだ！」
[悲しみ、ゆっくり]「もう、二度と会えないと思っていた…」
```

#### 2. 間をタグ付け
```
「あの年——（1 秒の間）—— 私はまだ若かった。」
```

#### 3. 多キャラのセリフ
```
A：なぜ戻ってきた？
B：（沈黙）……彼女と約束したから。
```

### ドラマ絵コンテ · 上級 prompt

ドラマ絵コンテでは、prompt が**カメラ言語**も両立する必要：

| ショットタイプ | Prompt キーワード |
|---|---|
| 大ワイド | `ワイド全景、広大な風景` |
| 中景 | `ミディアムショット、ウエストアップ、会話シーン` |
| クローズアップ | `クローズアップ、感情のディテール、浅い被写界深度` |
| 主観ショット | `POV ショット、一人称視点` |
| 追従ショット | `トラッキングショット、被写体追従` |
| ドリー | `ドリーイン / ドリーアウト、ドラマチックな展開` |

## 25. 多言語ローカライズ

Wonderverse のコアユーザーは**中国と日本**を跨ぎ、ローカライズ能力はプラットフォームの堀です。
本章では日 / 英 / 韓 3 大市場への進出の実戦ポイントを提供。

### ローカライズの 3 つの層

[図：ローカライズの 3 つの層]

| 層 | ツール | 時間コスト | 適したシーン |
|---|---|---|---|
| 字幕 | [字幕生成 / 翻訳](#10-字幕生成--翻訳) | 5 分 / 話 | 海外で水温感を見る |
| 吹替 | [AI 吹替](#11-ai-吹替翻訳) | 15 分 / 話 | 正式配信 |
| 文化 | 現地脚本家 + 画像生成のスタイル調整 | 数日〜 | 旗艦 IP |

### 中国語 → 日本語

**言語スタイル**：
- 主語を省略することが多い（一人称はあまり出さない）
- 文末敬語（です／ます）はビジネスシーンに適合
- ドラマは「だ」原型 + 擬音語豊富を好む

**吹替アドバイス**：
- ドラマ女主役 → 「明朗 / 元気」音色
- 商業広告 → 「知的 / 落ち着き」音色
- 「翻訳調」を避ける—翻訳稿を先に日本人にレビューしてもらう

**字幕スタイル**：
- フォント優先順位：`Yu Mincho`、`Hiragino Sans`、`Source Han Sans JP`
- 1 行 16 全角文字以下
- 横並びの英文混在を避ける

### 中国語 → 英語

**言語スタイル**：
- 能動態優先
- 短文 + リズム感（速い切り替えのショート動画に適合）
- 文化的なネタは**等価翻訳**が必要、直訳ではない

**吹替アドバイス**：
- 米国音色：若い / 明瞭 / 抑揚あり
- 英国音色：落ち着き / 高級感 / 高級品に適合
- 中立な国際アクセント：商業広告全般用

**字幕スタイル**：
- フォント：`Inter`、`Helvetica`、`Roboto`
- 1 行 42 文字以下（スペース含む）
- 全大文字を避ける（タイトル / CTA を除く）

### 中国語 → 韓国語

**言語スタイル**：
- 敬語階層が厳密（ドラマキャラ間の呼称が重要）
- ショートドラマは直接的な感情表現を好む

**吹替アドバイス**：
- ドラマ → 韓国式甘い / きりっと音色
- 商業 → 標準ソウル音

**字幕スタイル**：
- フォント：`Noto Sans KR`、`Apple SD Gothic`
- 行幅は日本語と類似

## 27. FAQ

本章では最頻出の 30 の質問をカテゴリ別にまとめました。あなたの質問が見つからない場合は [お問い合わせ](#29-お問い合わせ) までご連絡ください。

### アカウントと権益

**Q：複数人で 1 つのアカウントを共有してもいい？**
A：おすすめしません。チームを作成し、メンバーを招待してください。各人が独立したアカウントを持ちます。

**Q：Enterprise にアップグレードするには？**
A：右上 → アップグレード → チーム版 / Enterprise 版を選択〔フローは要確認〕。

**Q：無料枠はいくら？**
A：新規アカウントは 1000 ポイント贈呈〔パラメータ要確認〕、約 30 枚の画像 / 5 本の動画分。

### 生成品質

**Q：生成結果が満足できない時は？**
A：以下の順で試してください：
1. prompt を修正
2. パラメータ調整（解像度を上げる / スタイルプリセットを変える）
3. 参考画像を追加
4. ガチャ（2× / 4×）
5. [超解像強化](#9-超解像強化) で後処理

**Q：人物キャラがずっと変わってしまう、どうする？**
A：キャラを [パーソナルアセット](#22-パーソナルアセット) として保存、prompt で `@キャラ:名前` で固定。

**Q：動画の動きが硬すぎ / 激しすぎ？**
A：[動画生成](#8-動画生成) の「動き強度」パラメータを調整。

### 待ち時間

**Q：なぜキューが長い？**
A：ピーク時（夜 8 時 - 深夜）はキューが長くなります、**夜間バッチ**を設定してエージェントに自動スケジューリングさせるのがおすすめ。

**Q：バッチタスクをバックグラウンド実行できる？**
A：できます。送信後にページを閉じても実行は継続、完了時にメール通知。

### コラボと権限

**Q：プロジェクトを外部チームに共有するには？**
A：プロジェクト設定 → 公開リンク → 権限と有効期限を設定。

**Q：Enterprise アセットを暗号化できる？**
A：Enterprise 版はエンドツーエンド暗号化、プライベートデプロイ VPC 分離をサポート〔詳細要確認〕。

### データと著作権

**Q：アップロードした素材は学習に使われる？**
A：デフォルトでは学習に使われません。Enterprise 版はデータ分離契約を締結可〔条項要確認〕。

**Q：生成結果の著作権は誰のもの？**
A：有料ユーザーは商用著作権を保有〔詳細はサービス規約参照〕。

**Q：すべてのデータをワンクリックで削除できる？**
A：できます。設定 → プライバシー → アカウント抹消で全データ永久削除（30 日後復元不可）。

### 課金と請求書

**Q：ポイントは期限切れする？**
A：無料ポイント 30 日、有料ポイントはサブスクサイクルに従う〔ポリシー要確認〕。

**Q：請求書を発行できる？**
A：できます。詳細は [課金と請求書](#28-課金と請求書) を参照。

### 技術的な質問

**Q：ブラウザ要件？**
A：Chrome / Edge 最新版推奨。Safari は一部機能が制限される場合あり。

**Q：モバイル対応？**
A：モバイルは現在閲覧のみ、編集はデスクトップ〔プロダクト段階要確認〕。

**Q：対応するインポートファイル形式？**
A：
- 画像：JPG / PNG / WebP / TIFF
- 動画：MP4 / MOV / AVI / MKV
- 音声：MP3 / WAV / M4A
- テキスト：TXT / DOCX / PDF（台本）

## 28. 課金と請求書

Wonderverse は**サブスク + 使用量**のデュアルモードを採用、軽量ユーザーと重度チームの両方に対応。
本章では価格構造、ポイントルール、請求書申請フローを説明します。

### プラン構造

| プラン | 適したユーザー | 月額〔例〕 | 月間ポイント | チーム人数 |
|---|---|---|---|---|
| **Free** | 個人試用 | ¥0 | 1,000 | 1 |
| **Pro** | 独立クリエイター | ¥199 | 12,000 | 1 |
| **Team** | 中小チーム | ¥999 / チーム | 80,000 | 5 人〜、1 人追加で課金 |
| **Enterprise** | 大企業 | 営業に連絡 | カスタム | 無制限 |

> ⚠️ 上記の価格は例〔実際の価格は要確認〕、公式サイトを基準としてください。

### ポイント課金ルール

| ノード | 消費ポイント〔例〕 |
|---|---|
| 画像生成 · 高画質 | 5 / 枚 |
| 画像生成 · 超高画質 | 15 / 枚 |
| 動画生成 · 720p · 5s | 30 / 本 |
| 動画生成 · 1080p · 30s | 200 / 本 |
| 超解像強化 · 画像 | 3 / 枚 |
| 超解像強化 · 動画 | 1 / 秒 |
| AI 吹替 | 1 / 秒 |
| 字幕生成 / 翻訳 | 0.5 / 秒 |
| スマート消去 · 動画 | 2 / 秒 |
| 横→縦変換 | 0.5 / 秒 |

### ポイント照会と監視

- トップバーに現在の残ポイントを表示
- **アセットライブラリ → 課金** で使用カーブ / プロジェクトごとの消費を確認
- チーム管理者がメンバーごとの上限を設定可（個人の浪費を防ぐ）

### 自動更新 / 月額ルール

- 初回契約後は月額自動更新
- いつでも **設定 → サブスク** で自動更新を停止可
- 月額プラン内で使い切れなかったポイントは**翌月に持ち越されません**〔ポリシー要確認〕

### 請求書申請

1. **アセットライブラリ → 課金 → 請求書申請** に入る
2. 開票月 / 金額を選択
3. 請求書タイプを選択：
   - 増値税普通請求書（電子）
   - 増値税専用請求書（紙、郵送が必要）
4. タイトル、税務番号、メール / 郵送先を記入
5. 提出後 3〜5 営業日以内に発行

### 払い戻しポリシー

- 月額プラン：契約 7 日以内で実際にポイント未使用の場合、全額返金〔ポリシー要確認〕
- 年額プラン：月額換算
- 使用量超過分のポイントパック：販売後返金不可

### Enterprise 財務連携

Enterprise 版がサポート：

- 月次決算書
- API 使用明細エクスポート（CSV / Excel）
- 複数アカウント統合請求書発行
- 跨主体課金〔Enterprise カスタム〕
